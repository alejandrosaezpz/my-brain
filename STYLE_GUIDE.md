# Style guide

## Visual style

- Use clean GitHub Markdown.
- Keep headings consistent.
- Use short paragraphs.
- Use tables when useful.
- Use Java code blocks for code.
- Use checklists for quick review.
- Use badges consistently in the README.
- Use consistent status labels and difficulty labels.
- Use emojis only when they help scanning.

## Writing style

- Keep the tone professional and beginner-friendly.
- Prefer direct language.
- Avoid overcomplicated wording.
- Keep explanations study-friendly.
- Avoid long walls of text.
- Focus on what matters in interviews.

## Topic docs

- Start with the concept, then Java usage.
- Explain why it matters.
- Include methods or operations when relevant.
- Include a complexity table.
- Include common mistakes.
- Include interview patterns.
- End with a quick checklist.

## Problem writeups

- Write a short original summary.
- Explain intuition before code.
- Show the step-by-step approach.
- Include edge cases.
- Keep the solution explanation concise.
- Link related problems when helpful.

## Code style

- Use readable Java.
- Prefer standard library APIs.
- Use meaningful names.
- Keep methods small.
- Handle edge cases explicitly.
- Avoid clever one-liners.
- Prefer clarity over micro-optimizations.

## Status labels

Use these labels consistently:

- `Complete`
- `Partial`
- `Template only`
- `Not started`
- `N/A`

## Difficulty labels

Use these labels consistently:

- `Beginner`
- `Easy`
- `Easy-Medium`
- `Medium`
- `Medium-Hard`
- `Hard`
- `Mixed`
