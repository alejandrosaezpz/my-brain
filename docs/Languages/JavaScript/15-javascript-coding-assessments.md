# JavaScript Coding Assessments

## 1. What it is

A practical page for online coding tests where speed, correctness, and clean syntax matter.

## 2. Common HackerRank patterns

| Pattern | What shows up | Recommended approach |
| --- | --- | --- |
| Input parsing | lines, arrays, numbers, strings | use `split`, `map`, and careful casting |
| String manipulation | trimming, casing, tokenizing | normalize first, then process |
| Array traversal | sum, min, max, filtering | use loops or `reduce` |
| Frequency counting | duplicates, histograms | use `Map` or object |
| Sorting | ranking, ordering, top values | use comparator functions |

## 3. Common CodeSignal patterns

| Pattern | What shows up | Recommended approach |
| --- | --- | --- |
| Array transforms | change shape or values | `map`, `filter`, `reduce` |
| Matrix / grid work | rows, columns, coordinates | nested loops and boundary checks |
| Validation | format checks, rules, constraints | early returns and helper functions |
| Simulation | state updates step by step | keep the state simple |
| Lookup problems | complement or membership checks | `Map` / `Set` |

## 5. Interview tips

- read the prompt twice before coding
- choose a simple solution first
- name your variables clearly
- explain complexity out loud
- use built-ins only when they stay readable

## 6. Debugging tips

- log intermediate values early
- verify array indexes and loop bounds
- check mutating methods like `sort` and `reverse`
- inspect `undefined` vs `null`
- test empty input before submitting

## 7. Edge cases checklist

- empty input
- one-element input
- repeated values
- negative numbers
- zero and empty string
- large input
- already sorted input
- invalid or missing fields

## 8. Before submitting checklist

- [ ] solution handles empty input
- [ ] complexity is acceptable
- [ ] no accidental mutation
- [ ] variable names are readable
- [ ] sample tests pass
- [ ] edge cases were checked
